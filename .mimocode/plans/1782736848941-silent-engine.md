# Fix: Admin User Update Fails — MANYTOMANY Relationship Maps to PK Column

## Root Cause

In `_apply_parsed_to_obj` (factory.py:21), `_apply_parsed` (class_views.py:493), `_resolve_rel_keys` (factory.py:46, class_views.py:92), the code builds a `rel_fk_map` that maps relationship names to local FK columns. For MANYTOMANY relationships like `roles`, `rel_prop.local_columns` returns the source PK (`AdminUser.id`), creating mapping `{"roles": "id"}`.

When `MultiRelationWidget.parse()` returns `['["5"]']` (a list), the code sets `obj.id` to that list, causing:
```
UPDATE admin_users SET id=? WHERE admin_users.id = ?
parameters: (['["5"]'], 2) → type 'list' is not supported
```

## Fix

Skip MANYTOMANY relationships when building `rel_fk_map` in all 4 functions. Handle MANYTOMANY separately in update paths (create path needs different handling since objects aren't loaded yet).

### 1. `fastapi_console/views/factory.py`

- **`_resolve_rel_keys` (line 46)**: Skip MANYTOMANY — they can't map to a single FK column. Leave them in the resolved dict as-is (the caller `create_create_submit_view` passes them to `model(**parsed)` which will be handled by the `on_create` hook).
- **`_apply_parsed_to_obj` (line 21)**: Skip MANYTOMANY in `rel_fk_map`.
- **`create_edit_submit_view` (line 423-425)**: After `_apply_parsed_to_obj`, add MANYTOMANY handling using session to load related objects and set the relationship.

### 2. `fastapi_console/views/class_views.py`

- **`_resolve_rel_keys` (line 92)**: Skip MANYTOMANY.
- **`_apply_parsed` (line 493)**: Skip MANYTOMANY in `rel_fk_map`.
- **`_update_object` (line 522)**: Add MANYTOMANY handling after `_apply_parsed`.
- **`UpdateView.html_response` (line 654-658)**: Same MANYTOMANY handling after `_apply_parsed`.

### MANYTOMANY handling pattern

```python
from sqlalchemy import inspect as sa_inspect

mapper = sa_inspect(type(obj))
for rel_key, rel_prop in mapper.relationships.items():
    if rel_prop.direction != "MANYTOMANY":
        continue
    if rel_key not in parsed:
        continue
    value = parsed[rel_key]  # list[str] from MultiRelationWidget
    target_model = rel_prop.mapper.class_
    objs = []
    for pk in value:
        if not pk:
            continue
        try:
            loaded = await session.get(target_model, int(pk))
            if loaded:
                objs.append(loaded)
        except (ValueError, TypeError):
            pass
    setattr(obj, rel_key, objs)
```
