# Fix: Theme and UI Config Not Working

## Problem

The `theme` and UI config parameters (e.g., `sidebar_style`, `table_style`, `form_layout`, `ThemeConfig`) passed to `Admin()` are not reflected in the rendered admin panel.

## Root Cause

Two issues prevent the config from reaching the frontend:

### 1. Wrong template directory loaded

`Admin._init_jinja()` in `core.py:636` resolves templates from:
```
Path(__file__).parent.parent / "templates"  →  fastapi_admin/templates/
```

This is the **old template set** that doesn't consume theme/UI config. The **new, config-aware templates** are at the project root `templates/` directory but are never loaded.

**Evidence:**
- `fastapi_admin/templates/base.html` — minimal, no `admin_config`, `theme_css`, or `ui_config` references
- `templates/base.html` (root) — full theme support: `data-preset`, `theme_css` style injection, `admin_config.primary_color`, `ui_config.content_width`, etc.

### 2. Missing Jinja globals

Even if the correct templates were loaded, two required globals are missing from `_init_jinja()`:
- `admin_config` — the config dict (used by `base.html` for `admin_config.dark_mode_default`, `admin_config.primary_color`, etc.)
- `theme_preset` — the active preset name (used by `base.html` for `data-preset` attribute and by topbar for preset switcher highlighting)

## Plan

### Step 1: Replace `fastapi_admin/templates/` with root `templates/` content

Copy all files from `templates/` (root) into `fastapi_admin/templates/`, overwriting the old versions.

Files to replace:
- `base.html`
- `partials/topbar.html`
- `partials/sidebar.html`
- `pages/dashboard.html`
- `pages/list.html`
- `pages/form.html`
- `pages/login.html`
- `pages/audit_log.html`
- `pages/audit_detail.html`
- `pages/roles/` (new template directory)
- `macros/` (updated macros)
- New: `partials/flash_messages.html`, `partials/pagination.html`

### Step 2: Add missing Jinja globals in `core.py:_init_jinja()`

**File:** `fastapi_admin/admin/core.py`, around line 652

Add after the existing `theme_css`/`theme_font_import_url` globals:

```python
self._jinja_env.env.globals["admin_config"] = self.config.ui.apply_to_template_context()
self._jinja_env.env.globals["admin_config"]["admin_path"] = self.router.admin_path
if self.config.ui.theme:
    self._jinja_env.env.globals["theme_preset"] = self.config.ui.theme.preset
```

Also need to include the broader admin_config dict (not just UI) — merge `admin_config` from `_wire_app_state` into the Jinja env globals:

```python
# In _init_jinja, add:
admin_cfg = {
    "title": self.config.ui.title,
    "logo_url": self.config.ui.logo_url,
    "favicon_url": self.config.ui.favicon_url,
    "primary_color": self.config.ui.primary_color,
    "primary_color_dark": self.config.ui.primary_color_dark,
    "dark_mode_default": self.config.ui.dark_mode_default,
    "admin_path": self.router.admin_path,
}
if self.config.ui.theme:
    admin_cfg.update(self.config.ui.theme.to_context())
self._jinja_env.env.globals["admin_config"] = admin_cfg
```

### Step 3: Ensure `theme_preset` is always available

Add a default for when no theme is configured:
```python
self._jinja_env.env.globals.setdefault("theme_preset", "editorial")
```
