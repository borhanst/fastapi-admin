# Permission Management UI Refactoring Plan

## Goal
Replace the matrix-style permission grid with a multi-select widget. Add custom hooks to ModelAdmin for admin customization. Add direct permission widget to user form via generic CRUD.

## Part 1: ModelAdmin Customization Hooks

### 1.1 Wire Existing Validation Hooks
**File**: `fastapi_console/views/class_views.py`

Wire `validate_create()` and `validate_update()` (already defined in modeladmin.py but dead code) into the save flow after form parsing.

**File**: `fastapi_console/views/factory.py`

Same wiring for ViewFactory.

### 1.2 Add `process_form_data()` Hook
**File**: `fastapi_console/modeladmin.py`

```python
def process_form_data(self, data: dict, request=None) -> dict:
    """Process form data before save. Override for custom processing."""
    return data
```

Wire into class_views.py and factory.py after form parsing.

### 1.3 Add `save_model()` Hook
**File**: `fastapi_console/modeladmin.py`

```python
def save_model(self, obj, data: dict, request=None, is_create=False) -> None:
    """Custom save logic. Override for full control over save flow."""
    pass
```

When overridden, bypass default save logic and use this method instead.

### 1.4 Add Form Context Hook
**File**: `fastapi_console/modeladmin.py`

```python
def get_form_context(self, context: dict, obj=None, request=None) -> dict:
    """Customize form template context. Return modified context."""
    return context
```

Wire into class_views.py `_build_form_context()`.

### 1.5 Add Dynamic Field Hooks
**File**: `fastapi_console/modeladmin.py`

```python
def get_readonly_fields(self, obj=None, request=None) -> list[str]:
    """Return list of readonly field names. Override for dynamic readonly."""
    return self.readonly_fields or []

def get_hidden_fields(self, obj=None, request=None) -> list[str]:
    """Return list of hidden field names. Override for dynamic hidden."""
    return []
```

Wire into form field generation to filter readonly/hidden fields dynamically.

## Part 2: Permission Widget UI

### 2.1 New Alpine.js Component: `permissionWidget`
**File**: `fastapi_console/static/js/admin.js` (after `multiRelation` at line 203)

Add `Alpine.data('permissionWidget', ...)` with:
- Search & select tables via multi-select picker
- Expandable cards per table with action toggles (view/create/edit/delete)
- Toggle all/none buttons
- JSON serialization to hidden input `perm_data`

### 2.2 New Template Partial: `permission_widget.html`
